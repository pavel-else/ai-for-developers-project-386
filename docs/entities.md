# Сущности

## User

Человек. Может создавать встречи и бронировать чужие.

- id — уникальный идентификатор
- имя, email, часовой пояс
- может иметь несколько Schedule

## Schedule

Когда пользователь свободен. Расписание на неделю.

- id — уникальный идентификатор
- привязан к User
- набор Availability-правил (день недели + время начала/конца)
- может быть несколько (например, "рабочее" и "личное")

## EventType

Шаблон встречи, который хост публикует для бронирования.

- id — уникальный идентификатор
- привязан к User
- длительность
- название, slug (ссылка вида /user/slug)
- ссылка на Schedule, по которому генерируются слоты

## Booking

Факт бронирования.

- id — уникальный идентификатор
- какой EventType
- кто booker (User)
- кто host (User)
- время начала и конца
- статус (подтверждено, отменено, ожидает)
