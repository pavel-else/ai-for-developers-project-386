# Сущности

## User

Человек. Может создавать слоты, EventType и бронировать чужие слоты.

- id — уникальный идентификатор
- имя, email, часовой пояс

## EventType

Системный шаблон встречи. Только для чтения пользователями. Заполняется при инициализации.

- id — уникальный идентификатор
- название
- slug
- длительность

Начальные: "15 минут" (15), "30 минут" (30).

## Slot

Окно доступности, которое создаёт хост.

- id — уникальный идентификатор
- userId — чей слот
- startTime — начало окна
- endTime — конец окна

Может быть много. В одном Slot может быть несколько Booking, но они не пересекаются по времени.

## Booking

Факт бронирования.

- id — уникальный идентификатор
- slotId — в каком слоте
- startTime — начало встречи (≥ slot.startTime)
- endTime — конец встречи (≤ slot.endTime)
- bookerId — кто забронировал
- eventTypeId — по какому шаблону
- status — подтверждено, отменено, ожидает
