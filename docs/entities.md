# Сущности

## User

Человек. Все пользователи регистрируются (логин + пароль, без верификации).

- id — уникальный идентификатор
- имя, email, slug, часовой пояс

## EventType

Системный шаблон встречи. Только для UI выбора перед бронированием.

- id — уникальный идентификатор
- название
- длительность

Начальные: "15 минут" (15), "30 минут" (30).

## Slot

Окно доступности, которое создаёт host.

- id — уникальный идентификатор
- userId — чей слот
- startTime — начало окна
- endTime — конец окна

В одном Slot может быть несколько Booking, но они не пересекаются по времени.
Удаление слота блокировано, пока есть активные брони.

## Booking

Факт бронирования.

- id — уникальный идентификатор
- slotId — в каком слоте
- startTime — начало встречи (≥ slot.startTime)
- endTime — конец встречи (≤ slot.endTime)
- bookerId — кто забронировал
- duration — длительность встречи (15 или 30 мин)
- status — active / cancelled
